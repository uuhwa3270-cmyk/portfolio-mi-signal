KEY DECISIONS

핵심 의사결정 —
전략과 실행

25편의 리서치 문서를 기반으로, 서비스 방향과 품질을 결정했습니다

01 포지셔닝 전환

"편의성 도구" → "수요 창출 플랫폼"으로 서비스 컨셉 전환

경영진: "자동화는 기본값, 데이터로 지불의사를 만들어야 한다"

02 데이터 기반 설계 검증

36개 UI를 Snowflake 테이블과 1:1 대조 → 구현 가능성 58% → 100%

AI에게 시키기 전에 "이 데이터가 존재하는가"를 먼저 검증

03 Steering 자동화 체계

데이터 가이드 / 디자인 QA / Git 안전 규칙을 AI에 주입 → 반복 검증 자동화

규칙을 먼저 설계하고, AI가 그 규칙을 지키게 만든 구조

04 심리스 경험 설계

분석 → 판단 → 캠페인 집행까지 끊기지 않는 단일 흐름으로 통합

기존 자사 서비스의 경험 단절(페이지 분리) 문제를 해결

05 HTML 프로토타입 선행

Figma 없이 4,000줄 HTML로 먼저 시각화 → 실시간 검증 → React 전환

AI가 코드를 생성하더라도 "뭘 만들어야 하는지"는 사람이 먼저 정의

06 프롬프트 엔지니어링

단계적 프롬프트 설계 + 멀티 AI(GPT×Kiro) 교차 활용으로 정확도 확보

디자인 QA·기획 검증을 Steering으로 관리 + 단계적 프롬프트로 완성도 고도화
